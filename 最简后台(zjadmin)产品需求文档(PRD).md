# 最简后台\(zjadmin\)产品需求文档\(PRD\)

**文档版本**：v1\.0
**文档状态**：正式版
**编写日期**：2026\-06\-13
**技术栈**：\.NET Core 8 \+ Vue 3 \+ Vite \+ JavaScript
**系统名称**：最简后台 \(zjadmin\)

## 一、产品概述

### 1\.1 产品定位

zjadmin 是一套极简、轻量、开箱即用的通用后台管理系统，仅保留企业后台最核心的 RBAC 权限体系和操作日志功能，无冗余模块，可作为各类业务系统的基础权限框架快速二次开发。

### 1\.2 产品目标

- 提供完整的用户 \- 角色 \- 权限三级 RBAC 权限管理

- 记录所有用户操作行为和系统异常日志

- 代码结构清晰，易于理解和扩展

- 页面简洁美观，操作流畅

- 开箱即用，无需大量配置即可快速部署

### 1\.3 目标用户

- 中小型企业开发人员

- 快速搭建后台原型的产品团队

- 学习\.NET Core\+Vue3 技术栈的开发者

## 二、技术架构

### 2\.1 后端技术栈

- **框架**：[ASP\.NET](https://ASP.NET) Core 8 Web API

- **ORM**：Entity Framework Core 8

- **数据库**：支持 SQL Server/MySQL/SQLite \(默认 SQLite，开箱即用\)

- **身份认证**：JWT\(JSON Web Token\)

- **日志**：Serilog

- **接口文档**：Swagger/OpenAPI

- **依赖注入**：\.NET Core 内置 DI

- **权限验证**：基于策略的权限验证

### 2\.2 前端技术栈

- **框架**：Vue 3 \(组合式 API\)

- **构建工具**：Vite 5

- **UI 组件库**：Element Plus

- **路由**：Vue Router 4

- **状态管理**：Pinia

- **HTTP 请求**：Axios

- **图标**：Element Plus Icons

- **样式**：SCSS

## 三、功能需求

### 3\.1 登录模块

#### 3\.1\.1 登录页面

- 用户名 \+ 密码登录

- 记住密码功能 \(本地存储 7 天\)

- 验证码 \(可选，默认关闭\)

- 登录错误提示

- 登录成功后跳转至首页

- 自动登录 \(记住密码状态下\)

#### 3\.1\.2 退出登录

- 清除本地 token 和用户信息

- 跳转至登录页面

- 后端销毁 token

### 3\.2 首页

- 系统名称和 logo 展示

- 顶部导航栏：用户头像、用户名、退出登录

- 左侧侧边栏：根据用户权限动态显示菜单

- 主内容区：欢迎页面，显示系统基本信息

- 底部：版权信息

### 3\.3 用户管理

#### 3\.3\.1 用户列表

- 分页显示用户信息：ID、用户名、昵称、邮箱、手机号、所属角色、状态、创建时间、操作

- 搜索功能：按用户名、昵称、邮箱、手机号模糊搜索

- 筛选功能：按状态 \(正常 / 禁用\)、所属角色筛选

- 排序功能：按 ID、创建时间升序 / 降序

#### 3\.3\.2 用户操作

- **新增用户**：用户名 \(唯一\)、密码、确认密码、昵称、邮箱、手机号、所属角色 \(多选\)、状态 \(默认正常\)

- **编辑用户**：可修改昵称、邮箱、手机号、所属角色、状态 \(不可修改用户名\)

- **重置密码**：管理员可重置用户密码为默认密码 \(123456\)

- **删除用户**：软删除，删除后用户无法登录 \(不可删除超级管理员\)

- **禁用 / 启用用户**：禁用后用户无法登录

### 3\.4 角色管理

#### 3\.4\.1 角色列表

- 分页显示角色信息：ID、角色名称、角色标识、描述、创建时间、操作

- 搜索功能：按角色名称、角色标识模糊搜索

#### 3\.4\.2 角色操作

- **新增角色**：角色名称、角色标识 \(唯一\)、描述

- **编辑角色**：可修改角色名称、描述 \(不可修改角色标识\)

- **删除角色**：删除前检查是否有用户关联该角色，有关联则提示无法删除 \(不可删除超级管理员角色\)

- **分配权限**：为角色分配菜单权限和按钮权限

### 3\.5 权限管理

#### 3\.5\.1 权限列表

- 树形结构显示所有权限：权限名称、权限标识、权限类型 \(菜单 / 按钮\)、排序、操作

- 支持展开 / 折叠所有节点

- 支持搜索功能：按权限名称、权限标识模糊搜索

#### 3\.5\.2 权限操作

- **新增权限**：上级权限、权限名称、权限标识 \(唯一\)、权限类型 \(菜单 / 按钮\)、路由地址 \(菜单\)、组件路径 \(菜单\)、图标 \(菜单\)、排序、是否显示 \(菜单\)

- **编辑权限**：可修改除权限标识外的所有信息

- **删除权限**：删除前检查是否有角色关联该权限，有关联则提示无法删除 \(不可删除系统内置权限\)

### 3\.6 日志管理

#### 3\.6\.1 操作日志

- 分页显示操作日志：ID、操作人、操作模块、操作类型、请求地址、请求方法、请求参数、IP 地址、操作状态、操作时间、耗时

- 搜索功能：按操作人、操作模块、操作类型、IP 地址搜索

- 筛选功能：按操作状态 \(成功 / 失败\)、操作时间范围筛选

- 详情查看：查看完整的请求参数和响应结果

- 导出功能：导出 Excel 格式的操作日志

- 清理功能：可清理指定时间范围的操作日志

#### 3\.6\.2 异常日志

- 分页显示异常日志：ID、异常信息、异常类型、请求地址、请求方法、请求参数、IP 地址、操作人、发生时间

- 搜索功能：按异常信息、异常类型、操作人、IP 地址搜索

- 筛选功能：按发生时间范围筛选

- 详情查看：查看完整的异常堆栈信息

- 导出功能：导出 Excel 格式的异常日志

- 清理功能：可清理指定时间范围的异常日志

### 3\.7 个人中心

- 查看个人基本信息：用户名、昵称、邮箱、手机号、所属角色

- 修改个人信息：昵称、邮箱、手机号

- 修改密码：输入原密码、新密码、确认新密码

- 查看个人操作日志

## 四、非功能需求

### 4\.1 性能要求

- 页面加载时间：首屏加载时间≤2 秒

- 接口响应时间：90% 以上接口响应时间≤500ms

- 并发能力：支持 100 个并发用户同时在线操作

- 数据库查询：单表查询响应时间≤100ms

### 4\.2 安全性要求

- 所有 API 接口必须进行身份验证

- 密码采用 BCrypt 加密存储，不可明文存储

- 防止 SQL 注入、XSS 攻击、CSRF 攻击

- 敏感操作需要二次确认

- 记录所有用户操作和系统异常

- 登录失败 5 次后锁定账户 15 分钟

- JWT token 有效期为 2 小时，刷新 token 有效期为 7 天

### 4\.3 可用性要求

- 系统可用性≥99\.9%

- 支持主流浏览器：Chrome 90\+、Firefox 88\+、Edge 90\+

- 响应式设计，支持 1366×768 及以上分辨率

- 操作简单直观，用户无需培训即可使用

### 4\.4 可维护性要求

- 代码注释覆盖率≥30%

- 遵循统一的代码规范

- 前后端分离，便于独立开发和维护

- 模块化设计，易于扩展新功能

- 完善的日志记录，便于问题排查
